## 우선순위 (Top 5)

### P1. 핵심 플로우 테스트 도입 (Vitest + React Testing Library)
- **범위:** 렌더링, 다크모드 토글, read-only 모드, 공유 링크 생성, 상태 메시지 동작의 기본 테스트
- **예상 난이도:** 중
- **기대 효과:** 높음 (회귀 버그 방지, 리팩터링 안정성 확보)
- **예상 소요시간:** 6~8시간

### P2. 협업 연결 상태 UI 추가
- **범위:** y-webrtc 연결 상태(연결 중/연결됨/재연결 중)를 상단 배지 또는 상태 문구로 표시
- **예상 난이도:** 중
- **기대 효과:** 높음 (협업 시 사용자 혼란 감소)
- **예상 소요시간:** 4~6시간

### P3. 공유 링크 전략 개선 (압축/짧은 링크 fallback)
- **범위:** `md` 쿼리 길이 제한 대응(압축 또는 저장소 기반 토큰 fallback 설계)
- **예상 난이도:** 중~상
- **기대 효과:** 중~높음 (긴 문서 공유 실패 방지)
- **예상 소요시간:** 6~10시간

### P4. Export 품질 개선 (페이지 분리/스타일 안정화)
- **범위:** PDF 페이지 브레이크 개선, 코드블록/다이어그램 잘림 완화
- **예상 난이도:** 중
- **기대 효과:** 중 (실사용 문서 품질 향상)
- **예상 소요시간:** 4~6시간

### P5. README 사용 가이드 확장 (협업/공유/제약 사항)
- **범위:** 브라우저 권한, WebRTC 네트워크 제약, 공유 링크 사용법, 알려진 이슈 문서화

- **기대 효과:** 중 (온보딩 시간 단축, 문의 감소)
